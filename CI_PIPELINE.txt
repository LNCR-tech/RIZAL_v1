# CI Pipeline Summary — Aura
# File: .github/workflows/ci.yml

## Triggers

- `push`
- `pull_request`
- `workflow_dispatch`

`workflow_dispatch` also exposes a `deploy_frontend` boolean input so the frontend image can be published manually after all checks pass.

## Frontend Gate Order

### 1. `frontend-install`
- Runs on Node `20` and `22`
- Uses `npm ci` with retry
- Validates the dependency tree with `npm ls --all --omit=optional`
- Fails on missing packages, invalid peer deps, or dependency conflicts

### 2. `frontend-lint`
- Runs ESLint with `--max-warnings=0`
- Runs Prettier format check
- Uploads lint logs as artifacts

### 3. `frontend-typecheck`
- Validates required frontend env variables and URL shapes
- Runs `vue-tsc --noEmit`
- Uploads typecheck logs as artifacts

### 4. `frontend-security`
- Runs `npm audit --audit-level=high`
- Scans the frontend repo for hardcoded secrets and API keys
- Uploads security logs as artifacts

### 5. `frontend-test`
- Runs Vitest unit/component tests with coverage
- Uploads coverage and test logs as artifacts

### 6. `frontend-build`
- Runs on Node `20` and `22`
- Treats Rollup warnings as CI failures
- Builds the production bundle
- Validates `dist/` structure
- Enforces bundle-size thresholds:
  - JS per-file limit: `1.4 MiB`
  - CSS per-file limit: `150 KiB`
  - Total `dist/` limit: `8 MiB`
- Uploads build logs and `dist/` artifacts

### 7. `frontend-preview-check`
- Downloads the Node 20 production build artifact
- Serves the compiled bundle with `vite preview`
- Runs Playwright smoke checks for:
  - auth page render
  - exposed dashboard render + navigation
  - exposed workspace render + navigation
- Uploads preview logs and Playwright artifacts

### 8. `deploy-frontend`
- Runs only after all frontend gates and `docker-build` pass
- Publishes the frontend Docker image to GHCR
- Runs automatically on push to:
  - `integrate/pilot-merge`
  - `preproduction`
  - `production`
- Also runs on manual dispatch when `deploy_frontend=true`

## Backend / Full Stack Jobs

- `compose-config`
- `backend-checks`
- `backend-tests`
- `assistant-tests`
- `integration-tests`
- `e2e-tests`
- `docker-build`

`e2e-tests` now wait for the frontend preview gate so obvious frontend failures stop the pipeline before the full browser stack boots.

## Failure Behavior

- Deployment is blocked if any frontend job fails.
- GitHub Actions shows the exact failed stage in the job graph.
- Every frontend gate now writes a structured step summary (`GITHUB_STEP_SUMMARY`) with:
  - command run
  - exit code
  - parsed diagnostics
  - a tail excerpt of the failure log
- Frontend gates emit GitHub error annotations with file/line/column when parsable.
- Build, lint, typecheck, security, unit-test, and preview jobs upload logs + `ci-reports/` markdown summaries as artifacts.

## Reliability Notes

- Workflow concurrency cancels superseded runs on the same ref.
- Node dependency cache is enabled through `actions/setup-node`.
- Playwright browser cache is enabled for preview and full-stack E2E jobs.
- Seeder env creation is conditional so branches without `seeder/` do not fail on missing files.
